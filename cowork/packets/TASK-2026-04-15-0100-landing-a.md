# Goal

별도 경로(`/landing-a`)에 정보 허브 중심의 메인 랜딩 페이지를 추가한다. 구현 대상은 `frontend/app/landing-a/page.tsx`와 여기에 직접 필요한 로컬 스타일링 범위다.

기존 `frontend/app/page.tsx`는 건드리지 않는다.

UI 초안 참조 파일은 `cowork/drafts/isoser-landing-A.html`이다. 이 파일은 cowork draft이므로 실행 기준 문서는 본 packet이며, 구현 시에는 draft의 구조와 카피를 참고하되 저장소 실제 경로와 Next.js 구조에 맞게 변환한다.

이 페이지의 핵심 역할은 "국가 취업 지원 정보를 한곳에 모은 허브"라는 이소서의 정체성을 비로그인 사용자에게 직관적으로 전달하고, 가입 또는 프로그램 탐색으로 유도하는 것이다.

# User Flow

- 비로그인 사용자가 `/landing-a`에 접근한다
- 최상단 빨간 Ticker에서 마감 임박 프로그램이 좌로 흐르는 것을 본다
- 다크 Hero 영역에서 카피("취업 지원 정보, 흩어진 채로 두면 기회를 놓칩니다")와 D-day 카드 3개를 본다
- Hero CTA 두 개 중 하나를 선택한다: "AI 맞춤 추천 받기" → `/login`, "전체 프로그램 보기" → `/programs`
- 스크롤하면 검색바가 sticky로 상단에 고정된다 (nav 아래)
- 검색바: 키워드 입력 필드 + 카테고리 칩 필터 (전체/마감임박/AI·데이터/IT·개발/디자인/경영/창업/서울/경기/온라인/국비100%)
- 마감 임박 프로그램 섹션: 3열 카드 그리드. 각 카드에 출처, D-day, 태그, 관련도 매치바, "지원하기"/"비교추가" 버튼이 있다
- 광고 카드: 일반 카드와 동일 레이아웃이지만 "광고" 배지가 우상단에 붙고 테두리 색상이 다르다 (파란색 계열)
- 부트캠프 비교 분석 섹션: 다크 배경, 3개 프로그램을 나란히 비교. 1등 카드에 "나에게 가장 적합" 배지
- 이용 흐름 섹션: 4단계 (정보 탐색 → AI 맞춤 추천 → 서류 즉시 생성 → 바로 지원), 화살표 연결
- 하단 CTA 스트립: "프로필 등록하면 AI가 딱 맞는 것만 골라드립니다" + "Google로 시작하기" 버튼 → `/login`
- Footer: 로고 + 서비스 설명 + 저작권

# UI Requirements

- 폰트: Noto Sans KR + Syne (mockup 기준). 단, 프로젝트 전체가 Pretendard를 쓰고 있으므로 Pretendard로 대체해도 무방하다. Syne은 숫자/로고 강조에만 사용하며, 필요 시 font-weight로 대체 가능하다
- 색상: 다크 테마 기반 (--ink: #0A0F1E). mockup의 CSS 변수 체계 참고
- Ticker: 빨간 배경(#EF4444), 좌로 무한 스크롤 애니메이션(CSS keyframes), 높이 36px
- Nav: 다크 배경, sticky, backdrop-filter blur, 오른쪽에 오렌지 CTA 버튼
- Hero: 다크 배경, 그라디언트 glow 효과, eyebrow 배지(깜빡이는 dot), 큰 타이틀(그라디언트 텍스트), D-day 카드 스트립(3개 + "더 보기" placeholder), 버튼 2개
- 검색바: 흰 배경, sticky(top: nav 높이), 검색 input + 카테고리 칩 가로 스크롤
- 프로그램 카드: 3열 그리드, 긴급도별 테두리 색상(빨강/주황/초록), 관련도 매치바(프로그레스 바), 광고 카드 구분
- 비교 섹션: 다크 배경 라운드 박스, 3열 비교 카드, "나에게 가장 적합" 배지 (오렌지 배경), 관련도 퍼센트 바
- 이용 흐름: 흰 배경 라운드 박스, 4열 그리드, 각 단계에 번호 아이콘 + 화살표 연결
- CTA 스트립: 다크 그라디언트 배경, 좌측 카피 + 우측 오렌지 CTA 버튼
- 반응형: 모바일에서 카드 1열, 비교 카드 세로 스택, 이용 흐름 2열, 검색칩 가로 스크롤 유지

# Acceptance Criteria

1. `/landing-a` 경로에서 정보 허브 랜딩 페이지가 정상 렌더링된다
2. Ticker가 좌로 무한 스크롤되며 마감 임박 프로그램 목록을 보여준다
3. Nav가 스크롤 시 sticky로 상단에 고정된다
4. D-day 카드의 긴급도별 색상(빨강/주황/노랑)이 mockup과 일치한다
5. 검색바가 Nav 아래에 sticky로 고정된다
6. 프로그램 카드 그리드가 3열로 렌더링되고, 광고 카드에 "광고" 배지가 표시된다
7. 각 프로그램 카드에 관련도 매치바가 표시된다
8. 부트캠프 비교 섹션에서 3개 카드가 나란히 표시되고, winner 카드에 "나에게 가장 적합" 배지가 있다
9. 이용 흐름 4단계가 화살표로 연결되어 표시된다
10. "AI 맞춤 추천 받기", "Google로 시작하기" CTA 클릭 시 `/login`으로 이동한다
11. "전체 프로그램 보기" 클릭 시 `/programs`로 이동한다
12. 기존 `frontend/app/page.tsx`는 변경하지 않는다
13. 모바일(375px)과 데스크톱(1280px) 모두에서 레이아웃이 깨지지 않는다

# Constraints

- 기존 `frontend/app/page.tsx`를 건드리지 않는다. 별도 경로 추가만 허용한다
- 모든 프로그램 데이터(카드, D-day, 비교, Ticker)는 정적 더미 데이터다. 실제 API/DB 연동은 이 Task 범위 밖이다
- 관련도 퍼센트도 정적 하드코딩이다
- 검색바 입력과 칩 필터는 UI만 구현한다. 실제 필터링 로직 없음
- 비교 추가 버튼, 지원하기 버튼 등 카드 내 인터랙션은 UI만 구현한다. 실제 동작은 이 Task 범위 밖이다
- `use client` 지시어 필요 (칩 토글 등 클라이언트 상태가 있으므로)
- 새 경로 구현에 필요한 최소 범위에서만 파일을 추가한다. 공용 스타일/레이아웃 전역 수정은 피한다

# Non-goals

- 실제 프로그램 DB 연동
- 검색 및 필터 로직 구현
- 비교 기능 실제 동작
- 프로그램 카드 클릭 시 상세 페이지 연결
- 광고 시스템 연동
- SEO 메타데이터 최적화
- A/B 테스트 인프라 (랜딩 B와의 비교 라우팅 등)

# Edge Cases

- Ticker 텍스트가 화면보다 짧을 경우에도 매끄러운 무한 스크롤이 되어야 한다 (텍스트 2벌 반복으로 해결 — mockup 패턴 참고)
- 검색바 칩이 화면 너비를 넘칠 경우 가로 스크롤이 가능해야 한다 (스크롤바 숨김)
- D-day 카드의 "더 보기" placeholder는 dashed border 스타일로 비어있는 느낌을 준다
- 모바일에서 이용 흐름 화살표는 생략하거나 세로 방향으로 전환한다
