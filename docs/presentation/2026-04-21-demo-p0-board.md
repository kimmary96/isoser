## 단계별 blocker와 실패 가능성

| 단계 | blocker | 실패 가능성 | 대응 |
|---|---|---|---|
| 랜딩 | CTA가 확장 퍼널과 안 맞음 | 첫 클릭이 흔들림 | 첫 이동을 프로그램 목록으로 고정 |
| 프로그램 목록 | 발표용 프로그램이 안 보임 | 흐름 시작 실패 | 발표용 데이터 미리 고정 |
| 프로그램 상세 | 상세 정보 빈약 | 설득력 약화 | 설명 가능한 상세 1개 선정 |
| 비교 또는 북마크 | 다음 액션이 모호 | 로그인 전환 약함 | 비교 후 로그인 CTA 고정 |
| 로그인 | 세션/리다이렉트 불안정 | 퍼널 중단 | 발표용 계정 fallback 준비 |
| 온보딩 | PDF 파싱 실패 | 이후 전체 흐름 차질 | 샘플 계정 대체안 필수 |
| 추천 대시보드 | 추천 결과 약함 | 개인화 가치 약화 | 추천 잘 나오는 계정 확보 |
| 캘린더 일정 적용 | 선택한 프로그램 일정 반영이 안 보임 | 새 기능 가치가 안 보임 | 반영 결과가 눈에 띄게 보여야 함 |
| PDF 이력서 추출 | 추출 품질 편차 | 이후 성과 수정이 약해짐 | 샘플 PDF 고정 |
| 성과 수정 | 수정 UX 불분명 | 개입 가능 가치가 안 보임 | 수정 포인트 1~2개만 데모 |
| 이력서 생성 | 입력/선택 부족 | 마지막 산출물 약함 | 샘플 활동 세팅 |
| 포트폴리오 생성 | 생성 수준 불명확 | 과장 리스크 | 발표 표현 기준 먼저 고정 |
| 취업 준비 완료 | 종료 화면/메시지 부족 | 데모 마무리 약함 | 완료 상태를 한 문장으로 닫기 |

## 코드 수정 전 smoke test 체크리스트

1. `/landing-a`가 정상 진입하는가
2. 랜딩에서 프로그램 목록으로 바로 이동 가능한가
3. 발표용 프로그램이 목록에 안정적으로 보이는가
4. 프로그램 상세 정보가 충분히 설명 가능한가
5. 비교 또는 북마크에서 로그인으로 자연스럽게 이어지는가
6. 로그인 후 온보딩 또는 대시보드로 예측 가능하게 이동하는가
7. 샘플 PDF 업로드가 실제 성공하는가
8. 실패 시 샘플 계정으로 즉시 대체 가능한가
9. 추천 대시보드에 추천 결과와 이유가 보이는가
10. 선택한 부트캠프가 캘린더 일정에 반영되는가
11. PDF 이력서 추출 결과가 다음 단계로 이어지는가
12. 성과 수정 결과가 저장 또는 반영되는가
13. 이력서 생성이 마지막까지 끊기지 않는가
14. 포트폴리오 생성이 발표 가능한 수준으로 보이는가
15. 취업 준비 완료를 보여줄 마지막 상태가 있는가
