# 2026-04-21 발표용 계정 / 샘플 데이터 준비 시트

## 문서 목적

- 발표 전에 필요한 계정, 샘플 PDF, 프로그램 URL, fallback 경로를 한 곳에 모은다.
- 실제 리허설 전에 비어 있는 항목이 없는지 빠르게 확인한다.
- 팀원들이 같은 자산을 기준으로 발표와 점검을 진행하게 한다.

## 사용 원칙

- 민감 정보는 문서에 직접 적지 않고, 위치 또는 관리 담당자만 적는다.
- 발표 전날까지 최소 1회, 발표 당일 시작 전에 1회 다시 확인한다.
- 값이 없으면 빈칸으로 두지 말고 `미정` 또는 `준비중`으로 적는다.

## 1. 발표용 계정

## 10. 오늘 바로 해야 할 입력 순서

### Step 1. 계정

- [ ] 메인 계정 식별자 입력
- [ ] fallback 계정 식별자 입력
- [ ] 각 계정 담당자 입력

### Step 2. PDF

- [ ] 샘플 PDF 1 저장 위치 입력
- [ ] 샘플 PDF 1 기대 추출 항목 입력
- [ ] fallback PDF 필요 여부 결정

### Step 3. 프로그램

- [ ] 프로그램 A 이름과 URL 입력
- [ ] 프로그램 B 이름과 URL 입력
- [ ] 프로그램 C 필요 여부 결정

### Step 4. 경로

- [ ] 프로그램 상세 고정 URL 입력
- [ ] 비교 고정 URL 입력
- [ ] 포트폴리오 생성 실제 경로 입력

### Step 5. fallback

- [ ] 로그인 실패 대체안 입력
- [ ] PDF 실패 대체안 입력
- [ ] 추천 약할 때 대체 계정 입력
- [ ] 포트폴리오 생성 불안정 시 대체안 입력

## 11. 입력 완료 기준

- 메인 계정과 fallback 계정이 모두 적혀 있다.
- 샘플 PDF 1의 저장 위치가 적혀 있다.
- 프로그램 A/B URL이 적혀 있다.
- 발표 퍼널 주요 URL이 적혀 있다.
- 최소 3개의 fallback 대체안이 적혀 있다.
- 담당자가 비어 있는 항목이 없다.
