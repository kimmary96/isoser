# 2026-04-21 Demo Must 발표 범위표 + P0 체크리스트

## 발표 원칙

- 발표 퍼널은 아래 한 줄로 확정한다.
- Demo Must 기능만 발표 범위로 본다.
- 발표 전에는 리팩토링하지 않는다.
- 발표 전 작업은 모두 P0 안정화 기준으로 판단한다.
- 발표용 계정과 샘플 데이터를 먼저 준비한다.

## 확정 발표 퍼널

`랜딩 → 프로그램 목록 → 프로그램 상세 → 비교 또는 북마크 → 로그인 → 온보딩 → 추천 대시보드 → 부트캠프 선택 시 캘린더에 일정 적용 → PDF 이력서 추출 → 성과 수정 → 이력서 생성 → 포트폴리오 생성 → 취업 준비 완료`

## Demo Must 발표 범위표

| 단계 | Demo Must 기능 | 발표에서 보여줄 것 | 완료 기준 | 검증 방법 |
|---|---|---|---|---|
| 1 | 랜딩 | 제품 소개와 프로그램 탐색 진입 | 발표 시작 화면에서 다음 클릭이 명확함 | 랜딩 진입 후 3초 안에 다음 행동 설명 가능 |
| 2 | 프로그램 목록 | 프로그램 검색/탐색 | 발표용 프로그램이 목록에 안정적으로 노출됨 | 같은 조건으로 재조회해도 보임 |
| 3 | 프로그램 상세 | 프로그램 개요와 지원 정보 | 상세 핵심 정보가 비어 있지 않음 | 상세 페이지 2회 확인 |
| 4 | 비교 또는 북마크 | 사용자의 선택 과정 | 비교 또는 저장 뒤 다음 액션이 분명함 | 비교 또는 저장 후 로그인 동선 확인 |
| 5 | 로그인 | 공개 탐색에서 개인화 전환 | 로그인 후 예상 화면으로 이동 | 로그아웃 상태에서 3회 반복 |
| 6 | 온보딩 | 이력 기반 추천 시작 | PDF 업로드 또는 샘플 계정 진입 가능 | PDF 1회 + fallback 1회 |
| 7 | 추천 대시보드 | 개인화 추천 제시 | 추천 결과와 이유가 보임 | 추천 화면 재조회 확인 |
| 8 | 캘린더 일정 적용 | 부트캠프 일정 반영 | 선택한 프로그램 일정이 캘린더에 보임 | 선택 후 캘린더 반영 확인 |
| 9 | PDF 이력서 추출 | 기존 이력서 데이터 활용 | 샘플 PDF에서 필요한 정보 추출 | 동일 PDF 2회 테스트 |
| 10 | 성과 수정 | 사용자가 결과를 손볼 수 있음 | 수정 전후 차이가 보임 | 수정 후 반영 확인 |
| 11 | 이력서 생성 | 지원 문서 생성 | 마지막까지 끊기지 않고 생성 화면 도달 | 2회 반복 테스트 |
| 12 | 포트폴리오 생성 | 확장 산출물 생성 | 발표 가능한 최소 결과가 보임 | 생성 결과 2회 확인 |
| 13 | 취업 준비 완료 | 퍼널 종료 | 마지막 상태를 한 문장으로 닫을 수 있음 | 리허설 종료 멘트 확인 |

## 발표용 계정과 샘플 데이터 선준비 체크리스트

### 계정

- [ ] 발표용 메인 계정 1개 준비
- [ ] fallback 계정 1개 준비
- [ ] 로그인 직후 온보딩으로 가는 계정인지 확인
- [ ] 추천 결과가 잘 보이는 계정인지 확인
- [ ] 이력서 생성에 필요한 활동 데이터가 있는지 확인
- [ ] 포트폴리오 생성에 필요한 활동 데이터가 있는지 확인

### 샘플 데이터

- [ ] 발표용 프로그램 2~3개 URL 확정
- [ ] 비교용 프로그램 조합 확정
- [ ] 샘플 PDF 1개 확정
- [ ] 성과 수정 예시 1~2개 확정
- [ ] 이력서 생성용 활동 세트 확정
- [ ] 포트폴리오 생성용 활동 세트 확정

### fallback

- [ ] PDF 파싱 실패 시 바로 쓸 샘플 계정 준비
- [ ] 특정 프로그램이 안 보일 때 대체 프로그램 URL 준비
- [ ] 추천 결과가 약할 때 보여줄 대체 계정 준비
- [ ] 마지막 산출물 화면을 바로 열 수 있는 경로 준비
