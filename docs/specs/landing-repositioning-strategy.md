# 이소서 랜딩 재포지셔닝 전략

## 목적

이 문서는 이소서의 랜딩과 초기 대시보드가 아래 의도를 분명히 드러내도록 재정리한다.

- 이소서의 본체는 `이력서 툴`이 아니라 `국비·지역 지원 프로그램 통합 탐색 + 개인화 추천 + 지원 준비 연결 플랫폼`이다.
- 이력서/포트폴리오 기능은 별도 제품이 아니라, 추천 정확도와 지원 전환을 높이는 연결 장치다.
- 프로그램 참여 경험이 다시 성과 데이터로 저장되고, 이후 추천과 취업 준비에 재사용되는 순환 구조를 보여줘야 한다.

## 한 줄 정의

이소서는 흩어진 국비·지역 지원 프로그램을 한곳에 모으고, 내 이력과 활동 기반으로 맞춤 추천과 빠른 지원 준비를 연결하는 플랫폼이다.

## 랜딩에서 반드시 전달해야 할 3가지

1. 다른 곳에 흩어진 프로그램을 한곳에서 볼 수 있다.
2. 이력/활동을 등록하면 더 정확한 추천을 받을 수 있다.
3. 지원하려는 순간 필요한 이력서/포트폴리오를 빠르게 준비할 수 있다.

## 메시지 우선순위

### 1순위 메시지

`흩어진 국비·지역 지원 프로그램을 내 이력 기반으로 추천받는다`

### 2순위 메시지

`추천만 해주는 게 아니라 지원 준비까지 바로 연결된다`

### 3순위 메시지

`프로그램 참여 경험이 다시 성과로 쌓여 다음 추천과 취업 준비에 쓰인다`

## 랜딩 정보 구조

### 1. Hero

목적:
- 이소서가 정보 사이트가 아니라 `추천 + 지원 준비 플랫폼`임을 즉시 이해시키기

구성:
- 메인 카피
- 보조 설명
- CTA 2개
- 추천 흐름 미니 다이어그램

권장 카피:
- 메인: `국비·지역 지원 프로그램, 내 이력 기반으로 한 번에 추천`
- 보조: `이력과 활동을 등록하면 나에게 맞는 프로그램을 추천하고, 지원에 필요한 이력서·포트폴리오까지 빠르게 준비할 수 있습니다.`
- CTA 1: `내게 맞는 프로그램 추천받기`
- CTA 2: `프로그램 둘러보기`

### 2. 문제 공감 섹션

목적:
- 왜 정보 허브만으로는 부족하고, 왜 이력 데이터가 필요한지 설명

권장 포인트:
- 프로그램 정보가 여기저기 흩어져 있다
- 어떤 프로그램이 나에게 맞는지 판단하기 어렵다
- 지원하려면 문서를 다시 준비해야 한다
- 참여 후의 경험이 다음 기회로 연결되지 않는다

### 3. 추천 원리 섹션

목적:
- 사용자가 `왜 이력서를 넣어야 하는지` 납득하게 만들기

권장 설명:
- 입력 데이터: 이력, 활동, 관심 분야, 지역, 목표 직무
- 출력 결과: 맞춤 추천, 비교, 지원 우선순위
- 연결 가치: 입력 데이터가 많을수록 추천 정확도가 높아진다

### 4. 순환 플로우 섹션

목적:
- 이소서의 핵심 구조를 시각적으로 설명

표현 순서:
1. 프로그램 탐색
2. 이력/활동 등록
3. 맞춤 추천
4. 지원 서류 빠른 생성
5. 프로그램 참여
6. 성과 저장
7. 다음 추천/취업 준비 재사용

### 5. 지원 준비 섹션

목적:
- 이력서 기능이 메인이 아니라 `지원 전환 장치`임을 보여주기

권장 포인트:
- 프로그램마다 요구하는 이력서/포트폴리오 대응
- 저장된 활동 데이터 재사용
- AI 코치로 빠른 보완
- 제출 직전 문서 정리

### 6. 성과 축적 섹션

목적:
- 리텐션 논리를 명확히 보여주기

권장 포인트:
- 부트캠프/프로그램에서 한 활동을 저장
- 다음 프로그램 추천과 취업 지원에서 다시 활용
- 시간이 갈수록 내 데이터가 더 강한 추천 기반이 됨

### 7. KPI / 후기 보드

목적:
- 강사 피드백 반영
- 제품 신뢰도와 사업성 근거 강화

## Hero 카피 후보

### 카피 1

- 메인: `국비·지역 지원 프로그램, 내 이력 기반으로 추천받고 바로 지원하세요`
- 서브: `흩어진 프로그램 정보를 한곳에서 보고, 내 경험에 맞는 기회를 찾고, 필요한 문서까지 빠르게 준비합니다.`

### 카피 2

- 메인: `기회를 모으고, 나에게 맞게 고르고, 바로 지원하는 플랫폼`
- 서브: `이력과 활동을 등록하면 프로그램 추천 정확도가 올라가고, 지원할 때 필요한 문서도 즉시 연결됩니다.`

### 카피 3

- 메인: `프로그램 추천부터 지원 준비, 참여 성과 기록까지 한 흐름으로`
- 서브: `이소서는 정보 탐색에서 끝나지 않고, 다음 기회와 취업 준비로 이어지는 순환 구조를 만듭니다.`

### 카피 4

- 메인: `흩어진 국비·지역 프로그램을 내 이력과 연결해주는 추천 플랫폼`
- 서브: `추천만 해주는 서비스가 아니라, 지원과 기록까지 이어주는 지원 준비 워크플로우입니다.`

## KPI 보드 제안

주의:
- 아래 수치는 현재 운영 데이터가 없으면 가설/추적 예정으로 표기한다.

보여주기 좋은 지표:
- `누적 프로그램 조회 수`
- `이력 기반 추천 생성 수`
- `프로그램 비교 사용 수`
- `지원용 문서 생성 수`
- `프로그램 참여 후기 수`
- `프로그램 이후 면접/취업 후기 수`

UI 예시 문구:
- `3,240회 추천 생성`
- `812건 지원 문서 준비`
- `146건 프로그램 후기`
- `57건 면접/합격 후기`

## 후기 보드 설계

### 후기 유형

1. 프로그램 합격 후기
2. 프로그램 수료 후기
3. 이력서/포트폴리오 제출 후기
4. 프로그램 참여 후 취업 연결 후기
5. 활동 저장 후 다음 지원에 재사용한 후기

### 후기 데이터 필드 제안

- 제목
- 사용자 유형
- 지원한 프로그램명
- 사용한 기능 태그
- Before/After 요약
- 결과
- 작성일

### 태그 예시

- `맞춤추천`
- `프로그램비교`
- `AI코치`
- `이력서생성`
- `포트폴리오`
- `성과저장소`
- `수료후기`
- `취업연결`

```text
[헤더]
로고 | 프로그램 탐색 | 추천 받기 | 후기 | 로그인

[Hero]
국비·지역 지원 프로그램, 내 이력 기반으로 한 번에 추천
흩어진 프로그램 정보를 모아 보고, 내 경험에 맞는 기회를 찾고,
지원에 필요한 이력서·포트폴리오까지 빠르게 준비합니다.
[내게 맞는 프로그램 추천받기] [프로그램 둘러보기]

입력: 이력/활동/관심분야
→ 분석: 개인화 추천
→ 전환: 빠른 지원 준비

[문제 공감]

- 나에게 맞는지 판단이 어렵다
- 지원할 때 서류를 다시 만든다
- 참여 후 기록이 남지 않는다

[추천 원리]
내 이력과 활동을 등록하면
- 더 맞는 프로그램 추천

- 지원 준비 속도 개선

[순환 플로우]
프로그램 탐색
→ 이력/활동 등록
→ 맞춤 추천
→ 지원 서류 생성
→ 프로그램 참여
→ 성과 저장

[성과 축적]
이번 프로그램의 경험이
다음 프로그램 추천과 취업 준비의 입력이 된다

[KPI 보드]
추천 생성 수 | 문서 생성 수 | 후기 수 | 합격/면접 후기 수

[후기 보드]
프로그램 합격 후기
수료 후기
취업 연결 후기

[마지막 CTA]
내 상황에 맞는 프로그램을 찾고
바로 지원 준비까지 시작하세요
[추천받기]
```

## 네비게이션 우선순위 제안

- `프로그램 탐색`
- `추천 받기`
- `후기`
- `워크스페이스`

주의:
- `이력서 만들기`를 독립 주메뉴로 올리기보다 `추천 받기`와 `워크스페이스` 안에서 연결하는 편이 현재 제품 정체성과 더 맞다.

## 디자인 방향

- 첫 화면에서 카드형 프로그램 목록을 바로 메인으로 두기보다, `추천 플랫폼`이라는 정체성을 먼저 보여주고 그 아래에 프로그램 피드를 배치한다.
- 프로그램 리스트는 정보 허브 역할을 하되, 각 카드나 섹션에서 `추천받기`, `비교하기`, `지원 준비 시작` CTA가 반복적으로 보여야 한다.
- 후기/KPI 보드는 대시보드형 수치와 실제 사용자 스토리를 함께 배치해 제품 신뢰를 만든다.
