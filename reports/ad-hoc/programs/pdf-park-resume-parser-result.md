# PDF Park Resume Parser Result

- changed files: `backend/chains/pdf_activity_parser.py`, `backend/chains/pdf_parser_rules.py`, `backend/chains/pdf_parse_utils.py`, `backend/chains/pdf_sentence_scorer.py`, `backend/tests/test_pdf_chain.py`, `backend/tests/test_pdf_sentence_scorer.py`, `backend/tests/fixtures/pdf_files/park_backend_resume.pdf`, `backend/tests/fixtures/pdf_texts/park_backend_resume.txt`, `backend/tests/fixtures/pdf_texts/pm_portfolio_resume.txt`, `backend/tests/fixtures/pdf_expected/park_backend_resume.json`, `backend/tests/fixtures/pdf_expected/park_backend_resume_pdf.json`, `backend/tests/fixtures/pdf_expected/pm_portfolio_resume.json`, `docs/current-state.md`, `docs/refactoring-log.md`
- why changes were made: 박준호 PDF에서 `FoodRunner` 프로젝트가 프로필 경력으로 잘못 들어가고, `백엔드 개발자 (5인 팀: ...)` 표기에서 역할이 `PM`으로 오인될 수 있는 파싱 경계를 보강하기 위해 수정했다.
- preserved behaviors: 기존 경력/프로젝트 날짜 분리, MoneyFlow 소개/기여 분리, Gemini quota fallback, 김지원 PM 포트폴리오 템플릿 파싱 동작을 유지했다.
- risks / possible regressions: `개발` 키워드가 넓어졌기 때문에 짧은 역할명 오분류 방지를 추가했지만, 새로운 직무명 표현은 추가 scorer 보강이 필요할 수 있다.
- refactoring done: scorer를 `SentenceClassification` + `classify_activity_sentence()` 점수 기반 구조로 분리하고, 기존 호출부는 호환 래퍼로 유지했다. 이후 `pdf_parser_rules.py`를 추가해 섹션 헤더, 활동 타입 alias, 소개/기여 키워드, 역할명 rule table, metric pattern, scorer weight를 공통 관리하도록 분리했다. PDF 템플릿별 fixture를 추가하고, role-only 직무명 사전을 확장했으며, scorer reason을 `metric_signal`/`keyword_signal`/`role_only` 등으로 세분화했다. `ISOSER_PDF_PARSE_DEBUG_SCORER=1` 옵션으로 실제 파싱 중 scorer reason debug log를 남기고, expected JSON snapshot fixture를 parameterized test로 검증한다. 실제 박준호 PDF 바이너리 fixture를 추가해 PyMuPDF extraction부터 fallback 파싱 snapshot까지 e2e로 고정했다.
- follow-up refactoring candidates: debug log를 request id와 연결하고, snapshot update helper 스크립트를 추가하면 운영 추적과 fixture 갱신이 쉬워진다.
- verification: `pytest backend/tests/test_pdf_chain.py backend/tests/test_pdf_sentence_scorer.py -q`, backend `py_compile`, frontend `tsc --noEmit`, 실제 `이소서_시연용_이력서_박준호.pdf` fallback parse를 통과했다.
