# User Flow

**진입점 A — 빈 슬롯 셀 클릭**

- 비교 카드의 빈 슬롯 영역("＋ 프로그램 추가")을 클릭하면 모달이 열린다
- 모달은 "어느 슬롯에 추가할지"를 컨텍스트로 기억한다

**진입점 B — CTA 행 "프로그램 추가" 버튼 클릭**

- 빈 슬롯에 대응하는 CTA 행의 ghost 버튼 "＋ 프로그램 추가"를 클릭하면 동일 모달이 열린다

**진입점 C — 하단 추천 카드 "비교에 추가" 버튼**

- 기존 흐름 유지. 모달 없이 첫 번째 빈 슬롯에 직접 추가한다

**모달 내부 흐름**

1. 모달 상단: 제목 "프로그램 선택", 서브텍스트, × 닫기 버튼
2. 탭 2개: "찜한 프로그램" (기본) / "전체 검색"
3. **찜한 프로그램 탭**
   - 로그인 사용자: `program_bookmarks` 기반 북마크 목록을 표시. 이미 비교 슬롯에 추가된 프로그램은 "추가됨" 상태로 비활성화
   - 비로그인 사용자: "로그인하면 찜한 프로그램을 바로 불러올 수 있습니다" 안내 + 전체 검색 탭으로 유도
4. **전체 검색 탭**
   - 검색 인풋 포커스 자동 활성화
   - 입력 시 프로그램명 / 카테고리 / 기관명으로 백엔드 검색 API 호출 (debounce 300ms)
   - 검색 결과 최대 20건 표시
   - 이미 비교 슬롯에 추가된 프로그램: "추가됨" 상태로 비활성화
5. 프로그램 카드의 "+ 추가" 버튼 클릭 → 해당 슬롯에 프로그램 추가 → 모달 닫힘 → 슬롯 UI 갱신 + URL ids 업데이트
6. 배경 클릭 또는 ESC 키로 모달 닫기

# UI Requirements

**모달 컨테이너**

- `position: fixed`, 전체 화면 오버레이 (`rgba(10,15,30,0.55)` + `backdrop-filter: blur(4px)`)
- 모달 박스: `max-width: 560px`, `max-height: 82vh`, `border-radius: 16px`, 스크롤 가능
- 열릴 때 `fadeIn + translateY(12px → 0)` 애니메이션 (0.2s)

**모달 헤더**

- 제목(16px bold) + 서브텍스트(12px muted)
- 오른쪽 상단 × 닫기 버튼 (28px 원형, hover 시 빨간 계열)

**탭 바**

- "찜한 프로그램" / "전체 검색" 2탭
- 활성 탭: `color: var(--blue)`, `border-bottom: 2px solid var(--blue)`

**프로그램 카드 (모달 내)**

- 좌측: 출처(10px muted) + 프로그램명(13px bold) + 태그 (카테고리, D-day, 찜 여부, 지원율)
- 우측: "+ 추가" 버튼 (fire 계열, hover 시 채워짐)
- 이미 추가된 프로그램: 카드 전체 `opacity: 0.45`, 버튼 → "추가됨" 텍스트로 교체, 클릭 불가

**검색 인풋**

- `width: 100%`, `border: 1.5px solid var(--border)`, `border-radius: 8px`
- focus 시 `border-color: var(--blue)`
- placeholder: "프로그램명, 카테고리, 기관 검색..."

**빈 상태 (북마크 0건 / 검색 결과 0건)**

- 중앙 정렬 안내 텍스트 표시

# Acceptance Criteria

1. 빈 슬롯 클릭 및 CTA "프로그램 추가" 버튼 클릭 시 모달이 열린다
2. 모달이 열릴 때 대상 슬롯 인덱스가 컨텍스트에 저장되어, 선택한 프로그램이 해당 슬롯에 정확히 추가된다
3. 찜한 프로그램 탭에서 로그인 사용자의 북마크 목록이 표시된다. API 오류 시 "불러올 수 없습니다" 안내를 표시하고 앱이 중단되지 않는다
4. 비로그인 사용자가 찜 탭을 열면 로그인 유도 안내가 표시된다. 전체 검색은 비로그인에서도 동작한다
5. 전체 검색 탭에서 입력 시 debounce(300ms) 후 백엔드 검색 API를 호출하고, 결과가 최대 20건 표시된다
6. 이미 비교 슬롯에 추가된 프로그램은 모달에서 비활성화 상태("추가됨")로 표시된다
7. 슬롯이 3개 모두 채워진 경우, 빈 슬롯 영역이 없으므로 모달 진입점 자체가 노출되지 않는다
8. 모달에서 프로그램 선택 시 모달이 닫히고, URL의 ids 파라미터가 업데이트되고(`router.replace`), 슬롯 UI가 즉시 갱신된다
9. ESC 키 또는 배경 오버레이 클릭으로 모달이 닫힌다
10. 하단 추천 프로그램 카드의 "비교에 추가" 버튼은 모달 없이 첫 번째 빈 슬롯에 직접 추가하는 기존 동작을 유지한다 (기존 Task 흐름과 충돌 없음)
11. 모달이 열린 상태에서 배경 스크롤이 차단된다 (`body overflow: hidden` 처리)

# Constraints

- 이 Task는 TASK-2026-04-15-1100이 구현 완료된 이후 진행한다. compare 페이지는 현재 `frontend/app/(landing)/compare/page.tsx` 경로에서 `/compare`로 제공된다

- 전체 검색은 기존 `GET /api/programs` 또는 백엔드 `/programs` 엔드포인트의 `q` 파라미터를 활용한다. 신규 검색 엔드포인트를 만들지 않는다
- 모달 컴포넌트는 `dashboard/_components/modal-shell.tsx`를 재사용하거나 같은 패턴으로 신규 작성한다. 전혀 다른 모달 구현체를 도입하지 않는다
- URL 상태 관리는 TASK-2026-04-15-1100과 동일한 `router.replace` 방식으로 통일한다
- 슬롯 상태(`slotState`) 관리는 URL ids를 단일 소스로 사용한다. 별도 클라이언트 상태로 이중 관리하지 않는다

# Edge Cases

- **비로그인 + 찜 탭**: 빈 목록 대신 로그인 유도 메시지를 표시한다. 탭 클릭 자체는 가능해야 한다
- **북마크 API 실패**: 찜 탭에 "불러올 수 없습니다" 안내 표시. 전체 검색 탭은 독립적으로 동작한다
- **검색 쿼리 빈 문자열**: 전체 목록(최신순 20건)을 표시하거나 빈 상태를 표시한다. 에러를 발생시키지 않는다
- **네트워크 지연**: 검색 중 로딩 스피너 표시. 이전 결과가 새 결과로 교체되기 전에 깜빡임이 최소화되어야 한다
- **이미 3개 슬롯이 모두 찬 상태**: 빈 슬롯이 없으므로 모달 진입점(빈 슬롯 클릭, CTA 버튼)이 노출되지 않는다. 이 상태에서 추천 카드 "비교에 추가" 클릭 시 "비교 슬롯이 가득 찼습니다" 토스트 안내를 표시한다
- **동일 프로그램 중복 추가 시도**: 모달에서 "추가됨" 비활성화 상태이므로 클릭 자체가 방지된다. 추천 카드 경로로는 토스트 안내 표시
